II Push Notifications — Smoke Test

Exercises the six push methods on a test II canister: registers a Service Worker, subscribes with the VAPID public key, grants consent, then triggers notify_user and shows the resulting OS notification.

Configuration

1. Sign in with II

—

2. Subscribe this browser

Registers /service-worker.js, fetches the VAPID public key from the canister, and calls pushManager.subscribe(). The resulting subscription is sent to push_subscribe_device. 

—

3. Grant consent

Records that this origin can send you push notifications on this identity. Also writes the reverse principal → (anchor, origin) index used by notify_user.

4. Fire a notification

If the relay accepts the outcall the notification pops up in your OS tray within a few seconds. Non-2xx statuses appear in the canister log (dfx canister logs).
